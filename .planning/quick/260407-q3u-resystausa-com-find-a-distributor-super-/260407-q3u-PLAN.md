Read `resysta-backup/site/find-a-distributor/distributors.json` for the scraped data, then modify `resysta-backup/site/find-a-distributor/index.html`:

**1. Identify the SSF widget block to replace:**
The entire SSF widget lives inside the `<div class="wpb_wrapper">` between lines ~1434-1644 (from `<div id='ssf-dummy-blck'` through the contact form popups ending at the closing `</div>` before `</div></div></div></div>`).

Specifically replace everything from:
```html

```
...through to just before:
```html
</div>  <!-- closes wpb_wrapper -->
</div>  <!-- closes wpb_text_column -->
```

**2. Replace with static card HTML:**
Insert a self-contained block with:

a) A `<style>` block for distributor cards (scoped with `.distributor-cards` parent):
   - Use CSS Grid: 3 columns on desktop (min-width: 992px), 2 on tablet (min-width: 768px), 1 on mobile
   - Card style: white background, subtle border (#e0e0e0), border-radius 8px, box-shadow on hover
   - Green accent (#7baa20) for company name heading and icon accents
   - Font: inherit from page (already has "Open Sans" / system stack)
   - Each card: padding 24px, company name as h3, address/phone/website/email as p elements with font-awesome icons (fa-map-marker, fa-phone, fa-globe, fa-envelope — FA is already loaded by SSF CSS)

b) Container HTML:
```html
<div class="distributor-cards">
  <div class="distributor-grid">
    <!-- One card per distributor -->
    <div class="distributor-card">
      <h3 class="distributor-name">Company Name</h3>
      <div class="distributor-info">
        <p class="distributor-address"><i class="fa fa-map-marker"></i> Address</p>
        <p class="distributor-phone"><i class="fa fa-phone"></i> <a href="tel:...">Phone</a></p>
        <p class="distributor-website"><i class="fa fa-globe"></i> <a href="..." target="_blank">Website</a></p>
        <p class="distributor-email"><i class="fa fa-envelope"></i> <a href="mailto:...">Email</a></p>
      </div>
    </div>
    <!-- repeat for each distributor -->
  </div>
</div>
```

c) Only include info fields that have actual data (skip empty phone/website/email lines).
d) For phone numbers, wrap in `<a href="tel:...">` for click-to-call.
e) For websites, wrap in `<a href="..." target="_blank" rel="noopener">`.
f) For emails, wrap in `<a href="mailto:...">`.

**3. Remove SSF-specific CSS and JS references** that are no longer needed:
- Do NOT remove the font-awesome CSS (still needed for icons on cards)
- Remove or leave the SSF normalize/mega-superstorefinder CSS links (harmless if left, but can comment out)
- The inline SSF style block near lines 553-680 can be left (it's inert without the widget)

**4. Remove the SSF contact form popup HTML** (lines ~1580-1644 — the modernBrowserPopup and modernBrowserConatct divs). These require WordPress backend and are non-functional.
